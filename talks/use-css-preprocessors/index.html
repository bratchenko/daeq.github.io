CSS Preprocessors

Superset of CSS.

Compiles to CSS by external tool.

Why use it?

  • No reasons not to use.
  • Faster writing.
  • Faster reading.
  • Less copy-pasting.
  • Becoming industry standard (and you'll be asked about it on a job interview).

Imagine application code

  • All in single file.
  • With a lots of unnamed numerical and literal constants.
  • With a lots of copy & paste.
  • Without abstractions.

So why write CSS this way?

How CSS Preprocessors can help?

  • Variables
  • Operations and calculations
  • Inheritance
  • Mixins
  • Modules and importing

Variables

			$pageWidth: 1024px;
			$mainTextColor: #5C5C5C
			 
			body {
			    color: $mainTextColor;
			}
			.container {
			    width: $pageWidth;
			}

Variables

Useful to create and manage styleguides.

Much more useful when combined with other preprocessor features.

Operations and calculations

			$defaultTextSize: 14px
			 
			.h1 {
			    font-size: $defaultTextSize + 12;
			}
			.h2 {
			    font-size: $defaultTextSize + 8;
			}

Operations and calculations

Much more readable styles:

			.some_element {
			    width: 322px;
			}
		
			.some_element {
			    width: $pageWidth - $sizebarWidth - $defaultMargin;
			}

Nesting

			.article {
				a {
					font-color: $linkColor;
					text-decoration: underline;
					&:hover {
						text-decoration: none;
					}
				}
			}

Inheritance

			.error {
			    border: 1px #f00;
			    background-color: #fdd;
			}
			.seriousError {
			    @extend .error;
			    border-width: 3px;
			}

Mixins

			@mixin vendor-prefix($name, $argument) {
			  -webkit-#{$name}: #{$argument};
			  -ms-#{$name}: #{$argument};
			  -moz-#{$name}: #{$argument};
			  -o-#{$name}: #{$argument};
			  #{$name}: #{$argument};
			}
			p {
			  @include vendor-prefix(hyphens, auto)
			}

Modules and importing

			@import "reset";
			@import "styleguide";
			@import "layout";

How to start?

  • Read about SASS, LESS or/and Stylus
  • Rename .css to .less (.sass/.stylus).
  • Set up LiveReload (you should try it anyway).
  • Refactor old styles when you have time.

You can do it this week.
