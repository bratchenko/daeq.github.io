Dev Team Lead

  • It's not a position. It's a role in a project.
  • We already have it.
  • Usually it's assumed by PM and partially by QA.

Dev Team Lead responsibilities

  • Split user stories (requirements) into tasks for developers.
  • Guide task esimation process.
  • Assign tasks to developers (and reassign them when needed).
  • Make sure developers' tasks are completed properly.
  • Make sure user story is implemented when tasks are completed.

Let's give this role to one of the developers

Because:
  • Developers better understand how to divide user story into tasks for developers.
  • Developers better understand when specific task is complete.
  • Developers better understand who among them is apt for the task at the moment.

Advantages for PMs and QAs

  • No need to dive into implementation. Deal only with user stories (requirements) and bugs.
  • You have at least one person who knows all the parts of project's technical implementation (ideally). So you can ask them if you don't know who to ask.

Advantages for developers

  • More flexibility in arranging work.
  • Better understanding of a project on a higher level (not limited with a subset of tasks assigned to you by PM).
