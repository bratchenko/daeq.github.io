When do we need automation?

Typical situation

  • I'm new to civilization.
  • I'm in the city.
  • I need to travel 20Km.
Should I walk or should I use transportation?

Walk

  • I know how to do it.
  • I always used walk before.
  • It's only 4-5 hours.

Use transport

  • Need to learn how to use transport.
  • Need to find out which route to use.
  • Need money.
  • Some people say it's faster.

Automation is investment of time.

It should save more time than it takes.

Idea of Yeoman

Tools for better development.

  • Generators for automation of writing code.
  • Grunt for automation of actions.
  • Bower for dependency management.

Generators

  • Start project with appropriate generator.
  • Generate files and components.
  • Write your own generators.

Good for both frontend and backend.

Grunt

  • Develop with lightweight server.
  • Watch for changes.
  • Build for production.

Bower

  • Install/update/remove dependencies with single command.
  • Have list of all dependencies and versions.
  • Write your own packages.

Generator for Stuzo projects

  • No backend setup. Start developing in 2 minutes.
  • LiveReload. Never refresh browser again.
  • LESS. More clear styles. Faster to create and modify.
  • Autoprefixer. No need to remember browser prefixes.
  • And more...
  • And there's more to come.
